# 動画量産システム 全体設計

## ゴール

顔出し・声出しなしの解説チャンネルを、**1日複数本の制作が回る体制**で運用し登録者10万人を目指す。
動画生成AIは使わず、静止画＋カット割り＋表情差分の「紙芝居」スタイルに徹する
（コストほぼゼロ・破綻ゼロ・完全に再現可能）。

## 決定事項（このリポジトリの前提）

| 項目 | 決定 | 理由 |
|---|---|---|
| テーマ軸 | 身近な科学・雑学 | ネタ無限・時事非依存・LLM量産時の事実誤りリスクが低い・市場実証済み。`channel.yaml` で変更可能 |
| 見せ方 | 2キャラ掛け合い（質問役×解説役） | 解説系の鉄板。対話形式はLLM台本生成と相性が良い |
| 尺・戦略 | 8〜12分ロング＋ショート切り出し | 収益化に有利。各動画からショートを自動切り出して流入源に |
| 音声 | VOICEVOX（ずんだもん・四国めたん） | クレジット表記で商用可。概要欄クレジットは自動挿入 |
| LLM | Claude API（キーが無ければコピペ半自動） | ネタ出し・台本のみLLM。以降は決定的な自動処理 |

```
│ ytf ideas│→ │ ytf script│→ │  ytf voice    │→ │ (自動合成)  │→ │ ytf render  │→ │ out/     │
│ LLM企画  │   │ LLM台本   │   │ VOICEVOX合成  │   │ Pillowで   │   │ FFmpeg 1パス│   │ video.mp4│
│ backlog  │   │ YAML検証  │   │ timing.json  │   │ カット静止画│   │ ASS字幕焼込 │   │ shorts   │
└─────────┘   └──────────┘   │ =時間の真実   │   └───────────┘   │ BGMダッキング│   │ サムネ    │

                                                                                    └──────────┘
```

### 設計の核: 「台本YAML = 単一の真実」

- LLMが関与するのは **script.yaml を作るところまで**。以降の全工程（音声・字幕・映像・章立て・
  クレジット・ショート切り出し）は台本から**決定的に**生成される。再実行すれば同じものができる。
- 字幕・カット割りのタイミングは、合成された**音声WAVの実測長**から確定する（`timing.json`）。
  推測や手動調整は不要で、口と字幕が絶対にずれない。
- Whisperは字幕生成には使わず、**誤読検出のQC**（台本と書き起こしの突合）に回す。

### 台本スキーマ（`ytf/schema.py`）

```yaml
meta:            # タイトル・slug・タグ・サムネ文言
scenes:          # シーン = 章。title は画面見出し＆概要欄の章立てになる
  - id: hook
    short: true  # ← ショート切り出し対象
    cuts:        # カット = 1セリフ。emotion で表情差分、slide で図解カード
      - speaker: zunda
        emotion: surprised
        text: "空って[NASA|ナサ]も認める青さなのだ"   # [表示|よみ] で読み指定
        slide: {title: "...", bullets: [...]}
```

### 各ステージの実装

| ステージ | モジュール | 中身 |
|---|---|---|
| ネタ出し | `ideas.py` | プロンプトテンプレ + LLM → `ideas/backlog.yaml` にストック。重複除外・スコア管理 |
| 台本 | `scripting.py` | 企画→台本をLLM生成、pydanticで検証、失敗時はエラーを渡して自動リトライ |
| 音声 | `voice.py` | VOICEVOX HTTP API。ユーザー辞書自動登録、感情→スタイル切替、`timing.json` 生成 |
| 素材 | `compose.py` / `assets_gen.py` | 背景+見出し+図解カード+立ち絵(表情差分・話者ハイライト)をPillowで合成。同一見た目はハッシュで共有 |
| 組み立て | `build.py` | FFmpeg concat demuxer（静止画に時間指定）+ ASS字幕焼込 + BGMサイドチェインダッキング + ラウドネス正規化(-14LUFS)。Macでは videotoolbox でエンコード |
| メタデータ | `metadata.py` | タイトル・概要欄（**VOICEVOXクレジット自動挿入**・章タイムスタンプ自動生成）・タグ |
| サムネ | `thumbnail.py` | 背景+キャラ+縁取り大文字2段。文言は台本の `meta.thumbnail` |
| ショート | `build.py render_shorts` | `short: true` のシーンを縦1080x1920で再レイアウトして切り出し |
| QC | `qc.py` | 尺チェック + Whisperで誤読候補の洗い出し |

### LLMの2モード（`llm.py`）

- **api**: `ANTHROPIC_API_KEY` があれば全自動（モデルは `channel.yaml` の `llm.model`）
- **manual**: プロンプトをファイルに書き出し → claude.ai に貼る → 返答を `--response` で渡す。
  APIコストゼロで同じパイプラインが回る

## リスクと対策

- **事実誤り**: プロンプトで断定回避を強制 + 投稿前に台本レビュー（人間が読むのは台本だけで済む設計）
- **VOICEVOX誤読**: `[表示|よみ]` 記法 + ユーザー辞書 + Whisper QC の三段構え
- **チャンネルの同質化**: `channel.yaml` と `prompts/` を差し替えるだけで別テーマ・別キャラの
  チャンネルを同じパイプラインで並行運用できる
