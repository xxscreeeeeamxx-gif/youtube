# yt-factory 動画制作スキル

顔出しなし解説チャンネル「ふしぎ研究所」の動画を、ネタ→台本→素材→演出→ビルド→検証まで一貫して作る。
このファイルは**過去の会話なしで新しいセッション・別のモデルでも1本作り切れる**ことを目的にした完全な手順書。

## 環境セットアップ（毎セッション最初に）

```bash
cd /Users/sasakihideaki/Claude/Projects/Youtube
. .venv/bin/activate
export PYTHONPATH=.
export YTF_FFMPEG="$PWD/tools/ffmpeg" YTF_FFPROBE="$PWD/tools/ffprobe"
```

- Python 3.9 の venv（`eval_type_backport` 導入済みなので `X | None` 記法も動く）
- ffmpeg/ffprobe は `tools/` の静的バイナリ（arm64、martin-riedl.de のビルド）。
  無ければ https://ffmpeg.martin-riedl.de/ からダウンロードして tools/ に置き `chmod +x`
- VOICEVOX は**手動起動不要**。`ensure_engine` がアプリ内蔵エンジン
  （`/Applications/VOICEVOX.app/Contents/Resources/vv-engine/run`）をヘッドレス自動起動
- Pexels APIキーは `.pexels_key`（gitignore済み）から自動で読まれる

## 全体フロー（1本作る）

1. **ネタ**: `ideas/backlog.yaml` から選ぶ。新企画は**必ず複数案を出してユーザーに選ばせる**
   （案はbacklogに同形式で追記し、採用したら `status: used`）。前回提示して不採用だった案を再提示しない
2. **事実確認**: 人物・年代・数字が絡むネタは WebSearch/WebFetch で必ず裏取りする。
   自信のない数字は台本で「〜と言われている」とぼかす。人名の読みは `[原昌宏|はらまさひろ]` 形式で固定
3. **素材収集**（下記「素材調達」）→ **必ずコンタクトシートを作って目視確認**、合わないものは再取得
4. **台本**: `projects/<slug>/script.yaml` を直接書く（スキーマ = `ytf/schema.py`、語り口 = 下記）
5. **音声とアニメ同期**（下記「アニメ同期ループ」）
6. **ビルド**: `python -m ytf.cli make <slug>`
7. **検証**（下記「検証チェックリスト」）→ 問題があれば直して再ビルド
8. **コミット&プッシュ**（下記「コミット規約」）→ `open projects/<slug>/out/video.mp4` でユーザーに見せる

## 動画の型（確立済みフォーマット）

- 掛け合い: **春日部つむぎ（先生）× ずんだもん（ちょっとアホな生徒）**。定義は channel.yaml
- 構成: **茶番**（ずんだもんが今日のテーマで損する小芝居・30〜60秒）→ フック → OP（自動挿入）→
  本編2〜3章 → **開発秘話の章**（人物ドラマ。日本人発明者ネタが鉄板）→ 実用の章 →
  小ネタ（タイトル無しシーン）→ 締め。**まとめ・振り返りの章は作らない。茶番で終わらない**
- 締めの章は `title: ""`。行動を促す一言 → 逆説や余韻の一言 → 登録呼びかけ →
  最後は**「ご視聴、ありがとうございました！」を2人のデュエット**（`duet_with:`）で終わる
- **1本1テーマを深く掘る**。周辺の別テーマに広げるより、本題の別側面（バリアフリー機能・
  裏側の設備・論争・歴史・実用）を章として掘り下げる（escalator回の車椅子ステップ章が原型）
- **漫才モード（実験中・escalator回から）**: ずんだもん=ボケ、つむぎ=ツッコミ。各章に
  ボケ→即ツッコミの応酬を1〜2回。ボケは題材への勘違い・大げさな定義で、解説の流れを止めない長さに
- **各動画は単体で完結させる**。固定視聴者は少ない前提。過去動画を見た前提の言及は禁止:
  シリーズ名（「恩人シリーズ」等）・「前回」「◯◯に続く」・過去回で扱った人物や小ネタへの参照
  （「グッドイナフさんに並ぶ」「碁盤もフラッシュも」等）はすべてNG。比較や連想は動画内の情報だけで成立させる
- **論争の章**（題材にネット論争・俗説があれば1章立てる。cup-noodleの「体に悪いのか」が原型）:
  ずんだもんが「ネットで見たのだ」と俗説を持ち込む → つむぎが正面から受ける →
  実は事実/企業努力がすごい（本編の仕組みと伏線回収できると最高）→
  「本当の注意点は別にある」で実用的な着地。叩かれ点の擁護だけで終わらせず、
  本当のリスクを数字つきで示す（数字は「〜と言われている」でぼかす）
- 目標尺 9分前後（channel.yaml の target_length_minutes）

## 語り口（最重要）

- つむぎ: 語尾 〜よ／〜だよ／〜ね 系。ずんだもんを呼ぶときは必ず**ずんだもん**（キミ禁止）。
  視聴者への呼びかけは「あなた」
- ずんだもん: 語尾は必ず **〜なのだ／〜のだ**。一人称ボク。視聴者と同じ勘違いをして驚き・質問する役
- **1文を長めに**: 接続助詞（〜から、〜けど、〜して）で自然につなぐ。体言止め連発・断片禁止
- **誰でも一発で分かる表現だけ**: 強引な比喩・内輪スラング禁止。機能は擬人化せず動作をそのまま言う
- **前振り禁止**: 「ここが今日いちばん面白いところ」「ここからがかっこいい」等は書かず内容で語る
- **制作指示をセリフに混ぜない**（「ここが動画の核心」等はユーザー指示でありナレーションではない）
- かぎ括弧「」は使わない。感情は emotion（立ち絵の表情のみ変化。音声スタイルは常にノーマル固定・話速1.15）
- **多読み漢字は読みタグ必須**: 空(から/そら)等は `[空|から]` か言い換え。
  whisperでは同音誤読を検出できないので、心配な語は timing.json の moras で読みを直接確認

## 演出

- `telops:` 章の山場だけ（全体で8〜10個）。xl+glow は衝撃の事実・結論・数字のオチ用
- `stat:` 意外な数字のカウントアップ（半角%は自動で全角％化される）。アニメのキャプションと重ねない
- `slide:` 箇条書きカード（1枚3行・各行20字以内）。動画があるカットには置けない（動画が無効になる）
- 実写クリップ: `video:` + `video_full: true`（+ `video_span:`, `video_speed:`）
- **背景静止画だけの時間は2割まで**。残りは実写クリップ＋自作アニメで埋める
- 自作アニメ（Pillowフレーム→ffmpeg）はこのチャンネルの核。既存の生成スクリプトを参考に:
  `scripts/gen_qr_extras.py`（茶番演出・解剖ツアー・時代カード・成功/失敗演出が全部入りで一番参考になる）、
  `gen_battery_anim_duo.py`（フェーズ同期）、`gen_duo_extras.py`（時代カード原型）、`gen_infographics.py`
- 時代カード（年表バー＋人物）で開発秘話を魅せる。人物写真はWikimedia CommonsでCC0/CC BYを探す
  （出典とライセンスを必ず確認。「作者不明・出典がニュースサイト」はライセンス偽装を疑い使わない）。
  企業ロゴは商標なので使わず文字表記（例: DENSO）
- 全画面アニメは画面下部の左右（立ち絵の定位置）に重要な文字を置かない。
  見出しはy=150中央（左上の章タブを避ける）
- `duet_with: <キャラ>` で2人同時のセリフ（音声ミックス＋両立ち絵が明るくなる）
- BGM: シーンの `bgm:` で章・話題ごとに切り替え（assets/bgm/ の名前。未指定は前から継続、
  変わり目は自動フェード）。使い分けガイドは `assets/bgm/SOURCES.md`。
  タイトル無しシーンに分ければ章カード無しでBGMだけ切り替えられる
- SEは付けない（章切替のピアノは自動）

## アニメ同期ループ（自作アニメがある場合の手順）

自作アニメのフェーズはセリフの頭に同期させる。**尺が変わるたびにこのループを回す**:

1. 台本を書く（アニメを充てるカットに `video: assets/clips/<名前>.mp4` を先に書いてよい。
   voice はファイル不存在でも動く）
2. `python -m ytf.cli voice <slug>` → `projects/<slug>/audio/timing.json` ができる
3. timing.json から該当スパンのカット開始秒（相対）を計算し、生成スクリプトの
   フェーズ境界定数（P1S/P2S/…）と DUR（=スパン合計+1秒。ループ巻き戻り防止）に書く
4. スクリプト実行でアニメ生成 → `make <slug>`
5. **セリフを直したら必ず 2〜4 をやり直す**（境界がズレる）

実写クリップは「スパン合計 ≤ クリップ実尺 ÷ video_speed」を必ず確認（超えるとループして巻き戻る）。
足りないときは video_speed を下げる（スロー）か span を減らす。

## 検証チェックリスト（ビルド後に必ず）

- 尺: video.mp4 と audio/narration.wav の duration がほぼ一致（ffprobe）
- フレーム抽出で目視: 茶番演出 / 各アニメのフェーズ切替（セリフ頭と同期しているか）/
  テロップ・statの文字被りなし / 新規クリップ / 締めのデュエット（両立ち絵が明るい）
- `ytf qc <slug> --whisper`: フラグは大半が「隣カット混入＋同音字違い」の誤検出。
  **同音（満タン/マンタン等）なら無視、読みが本当に違う疑いだけ moras で確認**
- **助詞「は」の誤読スキャン（必須）**: 「〜はね、」は「ハネ（羽根）」に化ける
  （正解はね→セエカイハネ等）。timing.json の moras から `ハネ` を検出し、本文に
  羽/跳ねが無ければ `[〜はね|〜わね]` タグで修正。他の助詞パターンも同様に疑う
- **主張と行動の整合チェック**: 動画の結論（例: 2列で立つ）と、キャラの宣言や実用章の
  行動（例: 真ん中に立つ）が矛盾していないか、台本を通しで読んで確認する
- **左右・東西・大小など向きのある事実の検算**: 茶番や例え話に落とすときに反転しやすい
  （例: 東京=左に立つ→圧を受けるのは「右に立ったとき」）。事実確認メモと突き合わせる
- BGM切替: ビルドログの「BGM Nリージョン」と、境界時刻が章頭と一致しているかを
  `collect_bgm_regions` の出力で確認
- **ショートも必ずフレーム検証**: out/short_hook.mp4 から3枚以上抽出して目視。
  縦動画のv_fullは「幅フィット帯」表示（build.pyのvfit1）。図解・キャプションが
  切れていないか、立ち絵・字幕と帯が重なっていないかを確認する

  → assets/backgrounds/m_*.jpg（**コミットする**）
- **実写クリップ**: Pexels動画API（過去スクリプト参照: per_page=15, width≥1280, 尺チェック）
  → assets/clips/*.mp4（gitignore。**出典を assets/clips/SOURCES.md に必ず記録**）

  `scripts/export_zundamon_sprites.py` / `export_tsumugi_sprites.py`

## 便利コマンド

| コマンド | 用途 |
|---|---|
| `ytf edit <slug>` | ブラウザ編集画面（テロップ/読み/素材/BGM/書き出しがGUIで完結・port 8765） |
| `ytf voice <slug>` | 音声合成とtiming.json更新だけ（アニメ同期の実測用） |
| `ytf media "<query>"` / `--video` | Pexels/Wikimedia素材のDL |
| `ytf bgm "<雰囲気>" --name <名前>` | MusicGenでBGM生成 |
| `ytf dict add <表記> <よみ>` | VOICEVOX誤読の恒久修正 |
| `ytf qc <slug> --whisper` | 誤読チェック |
| `ytf op` | チャンネルOP再生成（導入→OP→解説 の位置に自動挿入） |

## 実装上の注意（ハマりどころ・全部実測済み）

- キャッシュの効き方: セリフ変更→そのカットの音声だけ再合成 / 演出・クリップ差し替え→
  該当セグメントだけ再レンダリング（クリップはmtimeで自動検知）/
  トランジションの見た目を変えたら build.py の `fade1` 等のバージョン文字列を上げてキャッシュを割る
- **ffmpegの映像+音声同一グラフに遅延SEを入れると無音になる**（スケジューリング起因・実測）。
  音声は必ず2パス（build.py実装済み）。触るときはこの構造を壊さない
- adelay には必ず `:all=1`（無いとステレオ片chが0秒に漏れる）
- drawtext に半角 `%` は Stray% で文字ごと消える（全角％に自動変換済み）
- ZIPの日本語ファイル名は cp932 で復元（`info.filename.encode('cp437').decode('cp932')`）
- 章トランジション中はナレーションに `lead`（1.8s）の無音。BGM切替・SEもこの頭に揃う
- スパンの先頭カットに video を置く。次の video 付きカットを span が飲み込むと後者が消える
- 縦shorts は hook シーン（short: true）から自動生成される
- モーラ単位の発話タイミングは timing.json の `moras`（読みの確認・語単位の演出に使う）
- 一時ファイルはスクラッチパッドではなく安定パスが必要なものだけ tools/ や assets/ に置く
